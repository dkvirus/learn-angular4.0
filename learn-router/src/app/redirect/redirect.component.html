重定向路由 （对应代码中的 app/redirect 目录）

一、什么是重定向路由？


1. 当我们在地址栏输入 "localhost:4200/home" 时，此时得到的 path 为 "home"，可以在上图中的路由表中找到对应的组件 HomeComponent，并显示该组件的内容，也就是我们看到的 首页内容。
2. 但是当我们输入 "localhost:4200" 时，此时得到的 path 为 ""，此时并没有对应的组件用来显示，我们的愿景是输入 "localhost:4200" 时，实际上让它显示到首页内容。
3. {{ "{ path: '', redirectTo: '/home', pathMatch: 'full' }" }} 就是做这么一件事，而这么一件事的专业术语就叫做 重定向。

二、使用方法： {{ "{ path: '', redirectTo: '/home', pathMatch: 'full' }" }}

1、属性 path = "" （空字符串） 表示 url = "localhost:4200" 时的情形。path = "home" 表示 url = "localhost:4200/home" 的情形，依次类推。
1、路由配置项中使用 redirectTo 指向需要重定向的地址。当 path = ""（空字符串）时跳转到访问 {{ "{ path: 'home', component: 'HomeComponent' }" }}。
2、设置 pathMatch 可能的值为 full 或者 prefix。大多数情况下设置为 full 即可。

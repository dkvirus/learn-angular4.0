2、模块的概念


组件化（把我们的项目以组件的形式构建）-> 模块化。在Node.js中，模块就是一个文件，引入一个文件是简单import、require(‘filePaht’)。 在我们的前端领域衍生出了很多的模块化的规范。我们有了很多类似于 AMD、CMD、ES6等等。从项目的组织（资源目录）而言我们通常能见到两种。

模块化一：按照类型分。比如将所有 html 页面放到一个文件夹下，将所有 js 文件放到一个文件夹下，这里存放 html 的文件夹叫一个模块，存放 js 的文件夹叫另外一个模块。
模块化二：按照功能分。用户管理功能包含有一个 html 页面和一个 js 文件，角色管理功能也包含一个 html 页面和一个 js 文件。将 html 页面和对应的 js 文件放到一个文件夹内。这个文件夹 只处理当前功能。

Angular 的模块化就是指模块化二：按照功能分的类型。这是一种面向对象的过程，而按照类型分是面向过程的处理方式。
